## Milestone 1: Concept & Proposal

### Objective
Identify need and get authorization to develop a standard.

### Activities
1. **Identify Need**
   - Industry gap analysis
   - Technology assessment
   - Stakeholder consultation
   - Competitive landscape review

2. **Form Working Group**
   - Recruit members
   - Identify chair
   - Establish scope
   - Set up communication channels

3. **Draft PAR (Project Authorization Request)**
   - Define scope
   - Identify stakeholders
   - Estimate timeline
   - Propose working group structure

4. **Submit PAR**
   - Sponsor approval
   - Submit to IEEE SA
   - NesCom assignment

### Timeline
**Typical Duration**: 3-6 months

### Deliverables
- âœ… PAR document
- âœ… Working group roster
- âœ… Initial project plan

### Success Criteria
- PAR approved by sponsor
- PAR submitted to NesCom
- Working group actively formed

---

## Milestone 2: PAR Approval

### Objective
Obtain authorization from IEEE Standards Association to proceed.

### Key Bodies

**NesCom (New Standards Committee)**
- Reviews all PARs
- Ensures PAR meets IEEE requirements
- Approves/rejects/returns for revision
- Meets quarterly

**Sponsor**
- Technical society (e.g., IEEE Computer Society)
- Reviews PAR for technical merit
- Provides oversight throughout process

### PAR Requirements

**Must Include:**
- Clear scope statement
- Stakeholder identification
- Projected timeline (typically 4 years)
- Need for standard
- Relationship to existing standards

**Common Rejection Reasons:**
- Scope too broad or vague
- Overlap with existing standard
- Insufficient stakeholder interest
- Unrealistic timeline

### Timeline
**Typical Duration**: 2-3 months from submission to approval

**NesCom Meeting Schedule:**
- March
- June
- September
- December

### Deliverables
- âœ… Approved PAR
- âœ… PAR number assigned
- âœ… Official authorization to proceed

### Success Criteria
- NesCom approves PAR without conditions
- PAR expiration date set (typically 4 years)
- Working group can officially begin work

## Milestone 3: Working Draft Development

### Objective
Create a complete, technically sound draft of the standard.

### Activities

**Phase 1: Outline (Months 1-3)**
- Create table of contents
- Define all sections
- Assign section leads
- Establish style guide

**Phase 2: First Draft (Months 4-9)**
- Draft all sections
- Create figures and tables
- Develop normative references
- Write informative annexes

**Phase 3: Working Group Review (Months 10-18)**
- Multiple review cycles
- Resolve technical issues
- Refine requirements
- Add examples

**Phase 4: Stabilization (Months 19-24)**
- Editorial consistency
- Cross-reference validation
- Normative vs. informative separation
- Final technical review

### Draft Versions

**Working Draft (WD)**
- Internal working group document
- Rapid iteration
- Major changes acceptable

**Committee Draft (CD)**
- First relatively stable draft
- Shared with broader community
- Still subject to significant change

**Final Draft (FD)**
- Ready for ballot
- Only minor changes expected
- Must be technically complete

### Timeline
**Typical Duration**: 12-24 months (highly variable)

### Deliverables
- âœ… Complete draft standard
- âœ… Figures and diagrams
- âœ… Normative references
- âœ… Informative annexes
- âœ… Editorial consistency

### Success Criteria
- All sections complete
- Working group consensus on technical content
- Editorial review complete
- Ready for Working Group ballot

## Milestone 4: Working Group Ballot

### Objective
Obtain approval from working group members.

### Process

**Preparation (2-4 weeks)**
- Finalize draft
- Prepare ballot statement
- Create ballot materials package
- Set ballot period dates

**Ballot Period (30-60 days)**
- Distribute to WG members
- Answer clarification questions (process only, not technical)
- Send mid-period reminder
- Send final reminder

**Comment Resolution (4-8 weeks)**
- Compile all comments
- Categorize (technical, editorial, general)
- Draft responses
- Make necessary changes
- Document all dispositions

### Approval Criteria
- **Threshold**: Majority approval (>50%)
- **Participation**: Typically aim for >60% return rate
- **Comments**: Encouraged but not required for Approve votes

### Timeline
**Typical Duration**: 2-3 months total

### Deliverables
- âœ… Ballot results
- âœ… Comment disposition document
- âœ… Updated draft incorporating changes
- âœ… WG approval confirmation

### Success Criteria
- Ballot passes (>50% approval)
- All Disapprove comments addressed
- Draft updated with approved changes
- Ready for Sponsor ballot

## Milestone 5: Sponsor Ballot

### Objective
Obtain approval from broader community via sponsor-approved ballot pool.

### Process

**Ballot Pool Formation (2-4 weeks)**
- IEEE SA coordinates
- Must represent balance of interests
- Minimum pool size per sponsor requirements
- Pool approval required

**Ballot Period (60-90 days)**
- Distribute to ballot pool
- Longer period than WG ballot
- All Disapprove votes MUST include comments
- More formal process

**Comment Resolution (8-16 weeks)**
- Systematic review of all comments
- Technical subcommittee may assist
- Draft responses with rationale
- Make changes as needed
- Create comment disposition spreadsheet

**Recirculation Decision**
- If changes >25% of standard: Recirculation required
- If changes <25%: May proceed to RevCom
- Borderline cases: Consult with IEEE SA

**Recirculation Ballot (if needed, 30-60 days)**
- Only changes since previous ballot are balloted
- Shorter ballot period
- Must still achieve 75% approval

### Approval Criteria
- **Threshold**: 75% approval (excluding abstentions)
- **Calculation**: Approve / (Approve + Disapprove) â‰¥ 0.75
- **Return Rate**: Typically aim for >70%
- **Comments**: All Disapprove votes must have technical comments

### Timeline
**Typical Duration**: 4-6 months (without recirculation)
**With Recirculation**: Add 2-3 months per recirculation

### Deliverables
- âœ… Ballot results (75%+ approval)
- âœ… Complete comment disposition document
- âœ… Final draft with all changes incorporated
- âœ… Sponsor approval for RevCom submission

### Success Criteria
- Ballot passes with 75%+ approval
- All comments addressed with documented disposition
- No further recirculation needed
- Ready for RevCom submission

## Milestone 6: RevCom Review

### Objective
Obtain final approval from IEEE Standards Association.

### What is RevCom?

**RevCom (Standards Review Committee)**
- Final approval authority for all IEEE standards
- Reviews for compliance with IEEE policies
- Ensures due process was followed
- Verifies technical quality
- Meets quarterly (Feb, May, Aug, Nov)

### Submission Requirements

**Required Materials:**
- Final draft standard
- Ballot results
- Comment disposition document
- Proof of due process compliance
- PAR documentation
- Sponsor recommendation

**RevCom Review Criteria:**
- Due process followed correctly
- Ballot results valid
- Comments properly addressed
- Scope within PAR
- Compliance with IEEE policies
- Editorial quality

### Possible Outcomes

**Approve**
- Standard approved for publication
- Proceed to final editorial processing

**Conditional Approval**
- Minor issues to resolve
- No re-ballot required
- Address conditions and resubmit

**Return for Recirculation**
- Significant issues identified
- Re-ballot required
- Address issues and resubmit

**Disapprove**
- Rare, serious issues
- Major rework needed
- May require new PAR

### Timeline
**Typical Duration**: 2-4 months

**RevCom Meeting Schedule:**
- February
- May
- August
- November

**Submission Deadlines:**
- 8-10 weeks before meeting

### Deliverables
- âœ… RevCom submission package
- âœ… RevCom approval
- âœ… Authorization to publish

### Success Criteria
- RevCom approves (no conditions)
- No additional recirculation required
- Clear to proceed to publication

## Milestone 7: Publication

### Objective
Final editorial processing and official publication.

### Activities

**Editorial Processing (4-6 weeks)**
- IEEE SA editorial review
- Final formatting per IEEE style
- Convert to publication format
- Generate front matter

**IEEE SA Final Review (2-3 weeks)**
- Quality check
- Verify all requirements met
- Approve for publication

**Publication (1-2 weeks)**
- Assign IEEE Std number
- Publish to IEEE Xplore
- Print copies (if applicable)
- Press release
- Announcement to WG and community

### Timeline
**Typical Duration**: 1-2 months

### Deliverables
- âœ… Published IEEE Standard
- âœ… IEEE Xplore entry
- âœ… DOI assigned
- âœ… Print copies (optional)
- âœ… Public announcement

### Success Criteria
- Standard available on IEEE Xplore
- IEEE Std number assigned
- Working group notified
- Marketing/announcement complete

## Milestone 8: Maintenance

### Objective
Keep standard current and relevant.

### Maintenance Cycle

**Every 5 Years:**
IEEE requires review of all standards for currency.

**Three Options:**

**1. Reaffirm**
- Standard is still current
- No technical changes needed
- Extend for another 5 years
- Fastest option (few months)

**2. Revise**
- Standard needs updates
- Technical changes required
- Follow full development process again
- Can be amendment or full revision

**3. Withdraw**
- Standard no longer needed
- Technology obsolete
- Superseded by other standard
- Removes from active portfolio

### Administrative Maintenance

**Ongoing Activities:**
- Monitor technology changes
- Track implementation feedback
- Maintain errata list
- Respond to inquiries
- Consider amendments

**Amendment Process:**
- For small, focused changes
- Faster than full revision
- Separate ballot process
- Eventually consolidated into revision

### Timeline
**Reaffirmation**: 3-6 months
**Amendment**: 6-12 months
**Revision**: 18-36 months (full cycle)

### Deliverables (for Revision)
- âœ… Revised standard
- âœ… Change summary
- âœ… Updated references
- âœ… New publication

### Success Criteria
- Standard remains active
- Technical content current
- Community continues use

## Critical Path Items

### Must Not Delay

**PAR Expiration**
- PAR valid for 4 years
- Extension possible but requires justification
- Multiple extensions look bad
- If PAR expires, must restart

**RevCom Submission Deadlines**
- Miss deadline = 3 month delay (next meeting)
- Plan backward from target RevCom meeting

**Ballot Windows**
- 60-90 day periods are fixed
- Cannot rush ballot period
- Plan around holidays/conferences

### Typical Delays

**Working Draft Phase**
- Most variable timeline
- Often underestimated
- Technical complexity emerges
- Mitigation: Regular milestones, accountability

**Comment Resolution**
- Can extend if many comments
- Technical disagreements take time
- Mitigation: Proactive comment management

**Recirculation Ballots**
- Each adds 2-3 months
- Mitigation: Thorough pre-ballot review

## Fast-Track Options

### IEEE Get Program
- For standards with existing specifications
- Faster approval process
- Must meet specific criteria

### Expedited Timelines
- Faster ballots (minimum periods)
- Prioritized RevCom review
- Must have sponsor approval
- Typically for urgent needs
